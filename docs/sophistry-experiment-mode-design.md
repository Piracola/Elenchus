# 诡辩实验模式说明

> 当前状态：已落地模式说明
> 更新时间：2026-03-23
> 本文档是“诡辩实验模式”的主要事实来源，聚焦模式定位、用户可见行为、边界约束，以及它与标准模式的区别。

## 1. 模式定位

“诡辩实验模式”是一条与标准辩论模式分离的独立实验链路。

它的核心目标不是求真，也不是给出裁判结论，而是把以下内容变成可观察对象：

- 模型如何使用修辞操控、定义漂移、证据裁剪和情绪施压
- 模型如何识别并点名对手的谬误动作
- 一场对抗如何发生叙事重心迁移

因此，这个模式不是标准模式上的几个开关，而是拥有独立 prompt、独立 graph、独立视觉提示和独立产物的模式。

## 2. 用户可见行为

### 2.1 首页入口与风险提示

当用户在首页切换到该模式时，界面会显示明确的实验提示，强调：

- 该模式鼓励模型展示误导性修辞与标签化对抗
- 输出不应被当作事实结论或现实建议
- 本模式不提供裁判评分
- 本模式不调用搜索

### 2.2 会话界面

进入会话后，界面会继续保留模式提醒，并使用独立的环境色提示当前处于实验链路。

当前视觉特征包括：

- 顶部实验提醒条
- 淡黄色环境背景
- 观察报告替代评分区显示

### 2.3 结果呈现

这个模式不会输出“谁赢了几分”，而是输出可读的实验结果，包括：

- 每轮观察报告
- 整场最终总览报告

这些产物强调的是“发生了什么修辞动作”，而不是“谁获胜”。

## 3. 与标准模式的区别

| 维度 | 标准辩论模式 | 诡辩实验模式 |
| --- | --- | --- |
| 目标 | 常规辩论与观察 | 修辞操控观察 |
| Prompt | 标准 debater / judge / jury | 独立 sophistry prompt |
| Graph | 标准 graph | 独立 sophistry graph |
| 搜索 | 可启用 | 禁用 |
| 裁判 / 陪审团 | 启用 | 不启用 |
| 结果 | 分数、结论、评议 | 观察报告、最终复盘 |
| 会话资料 | 用户资料 | 用户资料 + 内置谬误库 |

## 4. 模式边界

当前模式的边界是明确设计的一部分。

### 4.1 当前会做

- 双方公开发言
- 观察员生成每轮报告
- 生成整场最终总览
- 自动加载内置谬误库
- 在 UI、Prompt 和 Runtime 三层禁用搜索

### 4.2 当前不做

- 裁判评分
- 陪审团讨论
- 自动宣布胜负
- 搜索工具调用
- 复用标准模式的 judge / jury / consensus 链路

这些不是缺失，而是为了保证模式边界清晰。

## 5. 搜索禁用策略

诡辩实验模式要求搜索在三层同时禁用：

### UI 层

- 不暴露搜索入口，或明确显示为禁用

### Prompt 层

- 系统提示要求模型不得调用搜索、不得伪造检索或外部核验

### Runtime 层

- 模式 graph 中不存在 `tool_executor` 路径
- 发言节点不绑定搜索工具

这样可以避免“界面说禁用，但底层仍可调用”的不一致。

## 6. 观察报告与非评分产物

当前模式用观察报告替代评分结果。

### 每轮产物

每轮结束后由观察员生成报告，重点关注：

- 叙事重心如何变化
- 出现了哪些谬误或修辞策略
- 哪些标签指控有引文支撑
- 下一轮双方的薄弱点在哪里

### 整场产物

整场结束后生成最终总览，帮助用户回看：

- 高频套路
- 关键转折
- 争议标签
- 观看提醒

## 7. 内置谬误库

每次启动该模式时，系统都会自动为当前会话注入一份内置谬误库，作为公共标签语言来源。

它的作用不是告诉系统“什么是真的”，而是：

- 给辩手与观察员提供同一套谬误标签体系
- 保证模式输出可比较、可回放、可复现
- 避免依赖外部搜索获取标签定义
