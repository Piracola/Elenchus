# 主界面参考资料上传功能

## 功能概述

在主界面（辩题输入环节）新增了参考资料上传功能，允许用户在创建辩论会话之前准备好相关的参考文档。

## 功能特性

### 1. 上传位置
- **主界面**：在 `HomeView.tsx` 的辩题输入框下方添加了参考资料上传按钮
- **辩论界面**：原有的辩论界面（`ChatHeaderOverlay`）中的上传功能保持不变

### 2. 支持的文件格式
- 文件扩展名：`.txt`, `.md`, `.markdown`
- MIME 类型：`text/plain`, `text/markdown`, `text/x-markdown`, `text/md`
- 文件大小：最大 1 MB
- 文件编码：UTF-8, UTF-8 BOM, GB18030

### 3. 使用方式

#### 主界面上传流程：
1. 在主界面输入辩题
2. 点击右下角的"参考资料"按钮
3. 在弹出的窗口中：
   - 点击上传区域选择文件
   - 或者直接拖拽文件到上传区域
4. 可以上传多个文件
5. 点击"创建辩论"按钮时，所有参考资料会随辩论一起提交到后端

#### 辩论界面上传流程：
1. 创建辩论后，进入辩论界面
2. 点击顶部工具栏的"参考资料 N"按钮
3. 在弹出窗口中上传文件
4. 上传后会自动进行 AI 提炼并加入公共知识池

### 4. 两种模式的支持

#### 标准辩论模式
- 按钮颜色：默认主题色
- 功能完整：上传、查看、删除

#### 诡辩实验模式
- 按钮颜色：诡辩模式专属金色主题
- 功能完整：上传、查看、删除
- 注意：诡辩模式下搜索功能被禁用，但参考资料上传功能正常可用

### 5. 技术实现

#### 新增文件
- `frontend/src/components/home/HomeReferenceLibrary.tsx` - 主界面参考资料上传组件
- `frontend/src/components/home/HomeReferenceLibrary.test.tsx` - 单元测试

#### 修改文件
- `frontend/src/components/HomeView.tsx` - 集成上传组件
- `frontend/src/hooks/useSessionCreate.ts` - 返回 sessionId 以便后续上传文档

#### 工作流程
1. 用户在主界面选择文件（不立即上传）
2. 文件信息存储在 `pendingDocuments` 状态中
3. 点击"创建辩论"时：
   - 先创建会话
   - 获取 sessionId
   - 批量上传所有待上传的参考资料
   - 显示上传结果（成功/失败数量）
4. 后端自动处理：
   - 验证文件格式和大小
   - 存储原始文件
   - 调用 AI Agent 提炼结构化条目
   - 同步到会话的 shared_knowledge

### 6. 用户体验优化

- **拖拽上传**：支持拖拽文件到上传区域
- **批量上传**：可一次选择多个文件
- **文件列表**：显示已选择的文件名称和大小
- **删除功能**：可以移除不需要上传的文件
- **上传反馈**：创建辩论后显示上传成功/失败的数量
- **角标提示**：有待上传文件时在按钮上显示数量角标
- **错误处理**：文件格式错误或超大小会显示明确的错误提示

### 7. 与辩论界面的区别

| 特性 | 主界面 | 辩论界面 |
|------|--------|----------|
| 上传时机 | 创建辩论前预选，创建时上传 | 辩论进行中随时上传 |
| 存储方式 | 先存储在浏览器状态中 | 直接上传到后端 |
| 文件验证 | 前端验证 | 前端+后端双重验证 |
| AI 处理 | 创建辩论后统一处理 | 上传后立即处理 |
| 查看已上传文件 | 无（尚未创建会话） | 有（显示文档列表和状态） |

## 测试要点

1. **文件格式验证**：
   - 测试支持的文件格式能正常上传
   - 测试不支持的格式被正确拒绝

2. **文件大小验证**：
   - 测试 1 MB 限制是否生效
   - 测试空文件被拒绝

3. **批量上传**：
   - 测试多个文件能正确加入待上传列表
   - 测试重复文件的处理

4. **创建辩论流程**：
   - 测试没有参考资料时辩论能正常创建
   - 测试有参考资料时辩论创建+文档上传流程
   - 测试部分文档上传失败的情况

5. **模式兼容性**：
   - 测试标准辩论模式下的上传
   - 测试诡辩模式下的上传
   - 测试两种模式下的 UI 显示
