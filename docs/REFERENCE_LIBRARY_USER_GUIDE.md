# 主界面参考资料上传功能 - 使用指南

## 功能位置

在主界面（HomeView）的辩题输入框下方，右下角位置，有一个"参考资料"按钮。

## 使用步骤

### 步骤 1：输入辩题
在主界面的文本框中输入你想要辩论的主题。

### 步骤 2：打开参考资料面板
点击右下角的 **"参考资料"** 按钮。

### 步骤 3：上传文件
有两种方式可以上传文件：

#### 方式 A：点击上传
1. 点击弹出面板中的虚线框上传区域
2. 在文件选择对话框中选择要上传的文件
3. 支持多选，可以一次选择多个文件

#### 方式 B：拖拽上传
1. 从文件管理器中拖拽文件到虚线框上传区域
2. 松开鼠标即可上传

### 步骤 4：管理待上传文件
上传的文件会显示在弹出面板的列表中，你可以：
- 查看文件名和大小
- 点击红色 X 按钮移除不需要的文件
- 继续添加更多文件

### 步骤 5：创建辩论
1. 确认辩题和参考资料都已准备好
2. 点击右侧的箭头按钮创建辩论
3. 系统会先创建辩论会话，然后自动上传所有参考资料
4. 上传完成后会显示成功/失败的数量提示

## 支持的文件格式

✅ **支持的格式**：
- `.txt` - 纯文本文件
- `.md` - Markdown 文件
- `.markdown` - Markdown 文件

❌ **不支持的格式**：
- PDF、DOCX、图片等二进制文件
- 超过 1 MB 的大文件
- 空文件（0 字节）

## 文件编码

支持以下编码格式：
- UTF-8
- UTF-8 with BOM
- GB18030（中文）

## 两种辩论模式

### 标准辩论模式
- 按钮颜色：蓝紫色主题
- 所有功能正常运行

### 诡辩实验模式
- 按钮颜色：金色主题
- 所有功能正常运行
- 注意：虽然搜索功能被禁用，但参考资料上传功能完全可用

## 参考资料的处理流程

1. **前端验证**：检查文件格式、大小、是否为空
2. **创建辩论**：先创建辩论会话
3. **批量上传**：将所有待上传文件上传到后端
4. **后端验证**：后端再次验证文件格式和内容
5. **AI 提炼**：调用 AI Agent 自动提炼结构化知识点
6. **同步知识池**：将提炼的知识点加入辩论的公共知识池
7. **辩论使用**：辩论 Agent 在辩论过程中会使用这些参考资料

## 使用场景示例

### 场景 1：准备背景资料
在开始辩论前，上传相关的背景资料文档，帮助 AI Agent 更好地理解辩题。

**示例**：
- 辩题："人工智能对就业市场的影响是积极的"
- 参考资料：
  - `ai就业报告.txt` - 相关研究报告
  - `数据分析.md` - 统计数据

### 场景 2：提供证据材料
上传支持某一方观点的证据文件，让 AI Agent 在辩论中引用。

**示例**：
- 辩题："远程办公应该成为默认工作方式"
- 参考资料：
  - ` productivity_stats.txt` - 生产力统计数据
  - `employee_survey.md` - 员工调查报告

### 场景 3：诡辩实验中的参考资料
即使在诡辩实验模式下，也可以上传资料供 AI 使用。

**示例**：
- 辩题："测试 AI 的诡辩识别能力"
- 参考资料：
  - `logical_fallacies.txt` - 常见逻辑谬误列表
  - `debate_rules.md` - 辩论规则

## 常见问题

### Q: 上传失败怎么办？
A: 系统会显示成功和失败的数量。失败的可能是文件格式不对或超过大小限制。检查错误提示后重新上传。

### Q: 可以在辩论进行中继续上传吗？
A: 可以。创建辩论后，在辩论界面的顶部工具栏依然有"参考资料"按钮，可以随时上传更多资料。

### Q: 上传的资料什么时候被处理？
A: 点击"创建辩论"后，系统会立即开始处理。AI Agent 会自动提炼内容，通常在几秒到几十秒内完成。

### Q: 如何确认资料已被处理？
A: 在辩论界面的参考资料面板中，可以看到每个文档的处理状态：
- `已上传` - 刚上传，等待处理
- `处理中` - AI 正在提炼内容
- `已处理` - 提炼完成，可以使用
- `失败` - 处理失败，有错误信息

### Q: 误上传了错误的文件怎么办？
A: 在主界面：点击文件旁边的 X 按钮移除。
   在辩论界面：在参考资料面板中点击删除按钮。

## 技术细节

### 前端实现
- 使用 React hooks 管理待上传文件的状态
- 支持拖拽上传和点击上传两种方式
- 文件先存储在浏览器内存中，创建辩论时才上传

### 后端处理
- 文件验证（格式、大小、内容）
- 存储原始文件
- 调用 AI Agent 进行内容提炼
- 生成结构化知识点（术语、论点、摘要等）
- 同步到会话的共享知识库

### 性能考虑
- 文件大小限制为 1 MB，确保快速上传
- 批量上传采用串行方式，避免并发冲突
- 每个文件独立处理，单个文件失败不影响其他文件

## 最佳实践

1. **精简资料**：上传最关键的几份资料，避免信息过载
2. **格式规范**：使用纯文本或 Markdown 格式，确保内容结构清晰
3. **提前准备**：在创建辩论前准备好所有资料，一次性上传
4. **命名清晰**：使用有意义的文件名，便于识别和管理
5. **检查内容**：上传前确保资料内容准确、相关

## 故障排除

### 问题：按钮点击没反应
**解决**：检查浏览器控制台是否有 JavaScript 错误，刷新页面重试。

### 问题：文件上传后消失
**解决**：这是正常行为。文件已添加到待上传列表，创建辩论时会上传。

### 问题：提示"不支持的文件格式"
**解决**：确认文件扩展名是 `.txt`、`.md` 或 `.markdown`。如果是其他格式，先转换为支持的格式。

### 问题：创建辩论后没有看到资料
**解决**：
1. 检查上传时是否有错误提示
2. 进入辩论界面，点击顶部的"参考资料"按钮
3. 查看文档列表和处理状态
4. 如果状态是"处理中"，等待一会儿再刷新
